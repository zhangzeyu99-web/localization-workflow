# 游戏本地化质检工具 — 使用说明书

> **版本**: v3.0  
> **更新日期**: 2026-04-15  
> **适用阶段**: Post-AI-Translation（AI 粗翻后质检）  
> **当前主线能力**: 英语稳定，印尼语可用

---

## 一、这是什么

一个本地 Python 工具，用于对 **AI 粗翻后的游戏语言包** 做自动化质检。

**一个输入，两个输出**：

```
语言表 Excel ──→ 质检工具 ──→ 结果文件 result.xlsx（完整结果 + 需确认子集）
                    │                
 术语库 JSON ··→    │      ──→ 质检报告 report.xlsx（统计 + 错误模式 + 学习笔记）
  （可选）
```

---

## 二、快速开始

### 当前推荐方式：工作区批处理

推荐优先使用 `workspace_runner.py` 或 `cli.py`，而不是停留在旧版 GUI 的人工复制粘贴流程。

按工作区生成严格批次：

```bash
python workspace_runner.py --workspace "C:\Users\Administrator\Desktop\本地化处理" --project 土拨鼠 --lang auto --mode prepare --auto-fix --batch-size 80 --ai-scope all
```

批次回填后再合并：

```bash
python workspace_runner.py --workspace "C:\Users\Administrator\Desktop\本地化处理" --project 土拨鼠 --lang en --mode merge --auto-fix --strict-review
```

### 方式二：双击桌面快捷方式

桌面上有「游戏本地化质检工具」快捷方式，双击打开 GUI。

### 方式三：命令行

```bash
python process_language.py --input language.xlsx --auto-fix
```

### 环境要求

- Python 3.10+
- 安装依赖：`pip install -r requirements.txt`（只需 pandas + openpyxl）

### 当前完整链路

```text
语言表 / 术语表
  -> 结构预检
  -> 机审
  -> 自动修复
  -> 严格 AI 审核批次
  -> 严格 merge 回填
  -> 复核输出 result_{lang}.xlsx / report_{lang}.xlsx
```

## 三、当前主线能力

### 1. 严格 AI 审核链路

- `prepare / merge` 通过 manifest 和 fingerprint 绑定批次
- 模型回填必须逐条覆盖全部 ID
- 缺行、乱序、格式错误、输入漂移都会直接拒绝 merge

### 2. 拼音残留检测

- 可识别 `Hongshangu`、`Jushizhen`、`Meiguihu`、`Xigu...`、`Lanshidi` 这类专名残留
- 对已知地图名和地点名支持标准映射回写

### 3. 短文本长度预算

- 中文原文可见长度 `<= 10` 先进入候选池
- `mode=hard`：紧凑 UI / 按钮 / 标签
- `mode=soft`：普通短文本，只做软提示
- `mode=exempt`：编号专名、复杂富文本等豁免
- 原则是“自然可懂第一，长度第二”

## 四、GUI 使用方法

打开工具后只有一个界面：

### 操作步骤

1. **选择语言表 Excel** — 点「浏览」选择你的 AI 粗翻语言表
2. **术语库 JSON（可选）** — 如果有术语库就选上，没有留空
3. **勾选「自动修复」** — 建议勾选，脚本会自动修复能修的问题
4. **选择语言** — 默认英语
5. **点击「开始质检」** — 等待处理完成

### 处理完成后

界面会显示结果摘要，以及三个按钮：
- **打开结果文件** — 直接打开 result.xlsx
- **打开质检报告** — 直接打开 report.xlsx
- **打开输出文件夹** — 打开 output 目录

## 五、输入文件格式

### 语言表（必需）

Excel 文件，至少 3 列：

| 列 | 字段 | 示例 |
|----|------|------|
| A | ID | 1, 2, 3... |
| B | 原文 | 充值积分达到{icon1}[color=#dc6c00]{num1}[/color] |
| C | 译文 | Top-up Points reaching {icon1}[color=#dc6c00]{num1}[/color] in total |

多语言格式也支持：A=ID, B=原文, C=英语译文, D=英语备注, E=法语译文...

### 术语库（可选）

支持两种格式，选一种即可：

**格式 A：Excel（推荐，和语言表一样的格式）**

| ID | 原文 | 译文 |
|----|------|------|
| 1 | 机甲战士 | Mech Warrior |
| 2 | 宝石 | Gems |
| 3 | 避难所 | Shelter |

脚本会自动读取 Excel 并提取中英对照关系，格式标签会被自动去除。

**格式 B：JSON**

```json
{
  "机甲战士": "Mech Warrior",
  "宝石": "Gems"
}
```

术语库由你自己手动维护。质检时如果提供了术语库，会自动对照检查译文中是否用了标准术语。

## 六、输出文件说明

### 文件 1：result_{lang}.xlsx

包含两个 Sheet：

#### Sheet「完整结果」

与输入格式一致，多了「备注」列：

| ID | 原文 | 译文 | 备注 |
|----|------|------|------|
| 754 | 获得后，可解锁未来战士头像 | Unlocks the Austin Avatar upon obtaining | 句式统一: ... |
| 1202 | 首次攻占{cityname}时... | ...performance and took {num}... | 自动修复(variable_missing): ... |
| 800 | 普通文本 | Normal text | _(空)_ |

- 被修复的行：译文已替换，备注说明修改原因
- 无问题的行：原样保留，备注为空

#### Sheet「需确认」

需要你人工审阅的子集：

| 列 | 用途 |
|----|------|
| ID | 定位键 |
| 原文 | 中文参考 |
| 当前译文 | 当前的译文（可能已被自动修复过） |
| AI建议 | 脚本给出的建议 |
| 原因 | 为什么需要人工确认 |
| 置信度 | 脚本对自己判断的信心 |
| 是否UI | 是否被识别为UI文本 |

**你的操作**：在这个 Sheet 里审阅，如果需要改动，直接回到「完整结果」Sheet 修改对应行的译文即可。

### 文件 2：report_{lang}.xlsx

包含四个 Sheet：

| Sheet | 内容 |
|-------|------|
| 总览 | 统计数字（总行数/自动修复/需确认/无改动/UI文本数） |
| 错误模式 | 错误类型、数量、示例ID、中文描述 |
| 学习笔记 | 发现的语言规律（用于自我进化） |
| 详细记录 | 每条修改的完整记录：ID/原文/修改前/修改后/原因 |

## 六、质检规则

| 检查项 | 说明 | 自动修复 |
|--------|------|----------|
| 变量完整性 | `{icon1}`, `{num1}` 等是否缺失/多余 | 缺失的自动补到末尾 |
| BBCode标签 | `[color=xxx]...[/color]` 是否闭合、颜色值是否一致 | 颜色值不一致时修正 |
| 术语一致性 | 对照术语库检查标准术语是否使用 | 大小写问题可修正 |
| 句式一致性 | 同模板的译文必须用相同句式 | 高置信度时统一句式 |
| 中文残留 | 译文中是否残留中文字符 | 不自动修复，标记人工确认 |
| UI文本识别 | 短文本/按钮类文本自动识别 | 仅标记，不修改 |
| AI深度审校 | 预留接口，当前占位 | 后续接入 |

## 七、命令行参数

```bash
python process_language.py --input language.xlsx [选项]
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `--input` | 是 | 语言表 Excel 路径 |
| `--term-base` | 否 | 术语库文件路径（Excel 或 JSON） |
| `--lang` | 否 | 语言代码，默认 `en` |
| `--output-dir` | 否 | 输出目录，默认 `./output/` |
| `--auto-fix` | 否 | 加上此参数开启自动修复 |
| `--lang-index` | 否 | 多语言文件列索引（0开始） |

## 九、FAQ

**Q: 没有术语库怎么办？**  
A: 不选就行，术语检查会自动跳过，其他检查正常运行。

**Q: 多语言文件怎么处理？**  
A: GUI 中默认处理第一个语言列。命令行用 `--lang-index 1` 可以切换到第二个语言列。

**Q: 「自动修复」不勾会怎样？**  
A: 只检测不修改。所有问题都会记录在备注和报告中，但译文不会被改动。

**Q: 结果文件怎么用？**  
A: 打开 result.xlsx → 先看「需确认」Sheet 了解有哪些问题 → 回到「完整结果」Sheet 直接修改需要改的行 → 保存 → 这就是最终语言包。
