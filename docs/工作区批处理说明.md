# 工作区批处理说明

这个入口用于直接处理类似 `C:\Users\Administrator\Desktop\本地化处理` 这样的业务目录，不再手工逐个指定语言表和术语表。

## 目标

- 自动发现项目目录中的英语语言表
- 自动合并通用术语表和项目术语表
- 过滤 `~$` 临时文件和 `新建文件夹` 这类历史噪音目录
- 为每个项目生成独立输出目录

## 入口

文件：

- `workspace_runner.py`

命令示例：

```powershell
python workspace_runner.py --workspace "C:\Users\Administrator\Desktop\本地化处理" --lang en --project 土拨鼠 --auto-fix
```

自动检测项目中的英语/印尼语并分别处理：

```powershell
python workspace_runner.py --workspace "C:\Users\Administrator\Desktop\本地化处理" --lang auto --project 土拨鼠 --auto-fix
```

批量处理全部可识别英语项目：

```powershell
python workspace_runner.py --workspace "C:\Users\Administrator\Desktop\本地化处理" --lang en --auto-fix
```

## 当前识别规则

- 顶层目录按“项目目录”处理
- 顶层通用英语术语表会并入每个英语项目
- 顶层通用印尼语术语表会并入每个印尼语项目
- 项目内优先使用更完整、更靠近项目根目录的术语表
- 默认忽略：
  - `~$*.xlsx`
  - `新建文件夹`
  - `新建文件夹 (2)`

## 处理前预检

- 会自动检测语言表的源语言和目标语言
- 当检测结果与请求语言不一致时，会先打印预警
- `--lang auto` 时，会按检测到的目标语言分别处理可识别任务
- 当前预检主要覆盖：
  - `zh`
  - `en`
  - `idn`

## 输出

以 `土拨鼠` 项目为例，输出目录为：

- `C:\Users\Administrator\Desktop\本地化处理\土拨鼠\output_english`

输出文件包括：

- `result_en.xlsx`
- `report_en.xlsx`
- `term_base_merged_en.json`
- 归档子目录 `源文件名_en_时间戳`

## 已知边界

- 目前仍是“高质量规则引擎优先”，不是 API 驱动的全自动语义改写器
- 第一版重点是英语流程成熟化
- 当前剩余高频问题仍以术语漏判和句式不一致为主，后续应继续压缩：
  - 泛动词术语
  - 低质量术语表项
  - 句式一致性误报/漏报
