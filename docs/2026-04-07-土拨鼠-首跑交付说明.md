# 土拨鼠本地化首跑交付说明

## 1. 本次交付范围

本次交付针对 `C:\Users\Administrator\Desktop\本地化处理\土拨鼠` 的最新英语、印尼语语言表做了新的首跑，不复用旧 prompt、旧 response、旧回写结果。

本次首跑的目标是先把流程稳定性和覆盖率做对，再谈效率：

- 全量送审，不再只送部分机审问题行
- 小批次运行，降低上下文串流和批次失控风险
- 每个批次必须逐条覆盖全部 ID，禁止 AI 只返回“改动项”
- merge 只认 manifest，不再从 prompt 文本反解析 ID
- merge 前校验输入指纹，避免“准备阶段和回写阶段不是同一份语言表”

## 2. 已完成的代码改造

### 2.1 严格 AI 批次协议

核心文件：

- `C:\Users\Administrator\Desktop\localization-workflow-project\utils\ai_checker.py`

新增能力：

- 批次 prompt 改成强约束协议：
  - `ID | KEEP`
  - `ID | FIX | corrected translation`
- 强制 AI 对每个批次的全部 ID 逐条输出，顺序必须一致
- 为每个批次额外写出 `.json` manifest
- manifest 中保存：
  - 批次号
  - 批次类型（main / recheck）
  - 行数
  - 每个 row_id 的 fingerprint
- merge 时严格校验：
  - response 是否缺 ID
  - response 是否多 ID
  - 当前输入是否和 prepare 时是同一批词条

### 2.2 CLI 严格 prepare / merge

核心文件：

- `C:\Users\Administrator\Desktop\localization-workflow-project\cli.py`

新增能力：

- `--batch-size` 默认改为 `100`
- 支持 `--strict-review`
- prepare 时会清理旧 `ai_review` 目录下的旧批次文件
- prepare 时同时生成：
  - 主审批次
  - 术语二次复查批次
- merge 时直接读取 manifest 合并，不再扫描 prompt 文本猜 ID

### 2.3 工作区入口支持 machine / prepare / merge

核心文件：

- `C:\Users\Administrator\Desktop\localization-workflow-project\workspace_runner.py`

新增能力：

- `--mode machine`
- `--mode prepare`
- `--mode merge`
- `--batch-size`
- `--ai-scope`
- `--strict-review`

这样现在可以直接针对工作区目录批量跑，不需要回到旧 CLI 手工拼路径。

- `C:\Users\Administrator\Desktop\localization-workflow-project`

```powershell
python workspace_runner.py --workspace "C:\Users\Administrator\Desktop\本地化处理" --project 土拨鼠 --lang auto --mode prepare --auto-fix --batch-size 80 --ai-scope all
```

说明：

- `--ai-scope all`：全量送审，避免“导入词条过少”
