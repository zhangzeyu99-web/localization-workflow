# AGENTS.md

## 语言

- 始终使用简体中文回复。

## 本仓库定位

- 本目录是游戏本地化处理的主工作流根目录。
- 优先使用 `workspace_runner.py`、`cli.py`、`process_language.py`。
- 不要默认退回到旧版 GUI 的人工复制粘贴流程。

## 默认执行方式

- 当用户给出语言表文件或项目目录时，不要停在机审阶段。
- 除非用户明确要求只跑部分步骤，否则默认执行完整闭环：
  1. 预检
  2. 机审
  3. 自动修复
  4. AI 审核
  5. 严格回填
  6. 复检
  7. 导出最终版

## Subagent 规则

- 完整跑工作流时，默认启用 subagent。
- 按“语言”或“项目”拆分 subagent，不按同一语言表的批次拆分。
- 主 agent 负责总控、`prepare / merge`、最终回填和最终导出。
- 不允许多个 subagent 同时写同一个输出文件。
- 不允许把同一语言表的严格批次映射拆给多个 subagent 并发处理。

## 稳定性优先

- 首跑优先稳定性，不优先吞吐量。
- 默认使用小批次，建议 `batch-size=80~100`。
- `prepare` 和 `merge` 之间不要替换、重排或改写输入语言表。
- 如果发现输入漂移、批次错配、词条漏回、响应格式错误，优先保证链路正确，不要强行合并。

## 输入假设

- 当前主支持场景是：中文原文 -> 目标语言列。
- 当前已稳定或可用的目标语言包括：
  - `en`
  - `idn`
  - `fr`
  - `de`
  - `tr`
  - `es`
  - `pt`
  - `ru`
- 如果同目录或工作区里存在术语表，默认一起使用。
- 如果没有术语表，则按无术语模式继续处理，不要因此停住。

## 质量要求

- 变量、占位符、BBCode、换行必须严格保留。
- 不要为了缩短文案破坏占位符、标签、变量顺序或结构。
- 短文本长度规则默认启用：
  - 中文原文可见长度 `<= 10` 先入池
  - `mode=hard`：紧凑 UI / 按钮 / 标签
  - `mode=soft`：普通短文本软提示
  - `mode=exempt`：编号专名、复杂富文本等豁免
- 自然可懂优先，长度第二。

## 交付要求

- 结果直接输出回原目录或项目目录。
- 默认同时生成：
  - 最终版语言表
  - `result_{lang}.xlsx`
  - `report_{lang}.xlsx`
- 回复用户时给出最终可用文件的准确路径。
- 不要只交付中间结果，除非用户明确要求。
- 如果还有剩余问题，继续收口到可交付版本，不要停在“待确认”。
